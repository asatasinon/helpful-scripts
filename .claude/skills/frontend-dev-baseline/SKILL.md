# Frontend Dev Baseline

本 skill 用于在 `baby-growth-record-book` 仓库内执行前端开发时，统一“怎么做才不返工”。

先阅读 [references/frontend-constraints.md](./references/frontend-constraints.md)，再进入实现。

## 适用范围
- 微信小程序（`frontend/miniapp/`）
- H5 分享/报告页（`frontend/web/`）
- 管理后台（`frontend/admin/`）
- 前端共享组件（`frontend/shared/ui/`）

## 默认工作流

1. 先定页面域和上下游。
   明确当前需求属于小程序、H5 还是后台；确认依赖的是 `T06`（核心接口）、`T07`（汇总/趋势/提醒）还是 `T08`（AI/导出/分享）。

2. 先对契约，再写页面。
   读取 `D03` 与 `S01` 中对应接口，确认请求字段、响应壳、错误码、分页参数和幂等头是否匹配，避免先写死再返工。

3. 先搭状态骨架，再填业务交互。
   统一维护登录态、`family_id`、`baby_id`、时区与加载状态；除登录外，不允许在业务请求中“丢家庭上下文”。

4. 先覆盖异常态，再做视觉细化。
   先实现空态、错误态、无权限态、过期态与重试，再补充样式细节；业务可用性优先于视觉精修。

5. 变更后做联调自检。
   对照 `Q01` 至少检查：多家庭隔离、角色可见性、核心记录链路、AI 免责声明、导出重试与下载链路。

## 实现准则

## 依赖与工程
- 使用 `pnpm` 进行安装、脚本执行、锁文件管理。
- 不引入 `npm`/`yarn` 锁文件与命令。
- 新增前端子工程时，保持与现有目录边界一致，不把后端逻辑放到前端目录。

## 接口与数据
- 所有 API 调用按 `/api/v1` 约定组织。
- 所有响应按 `{ code, message, data }` 解析，不跳过 `code` 直接信任 `data`。
- 写接口在可重试场景下带 `X-Idempotency-Key`。
- 事件相关页面遵循 `event_type + payload` 模式，不把类型字段拍平成不可扩展结构。

## 权限与安全
- 前端只做“体验层”权限控制，禁止把关键安全逻辑下沉到前端判断。
- 涉及分享、AI、导出入口时，明确过期态和失败态文案。
- 日志与埋点禁止输出 token、手机号明文和身份敏感字段。

## 时间与展示
- 输入/展示时间默认考虑家庭时区；与后端交互统一使用 UTC 毫秒时间戳。
- 趋势与汇总页面默认支持 `7/30/90` 天窗口与自定义区间能力。

## 输出格式（执行结果回复）
完成开发后，输出建议包含：
1. 变更文件清单（按小程序/H5/后台分组）
2. 契约对齐说明（引用对应接口）
3. 异常态与权限态覆盖说明
4. 验证结果与未完成风险

## 触发示例

**示例 1**
输入：`把小程序快速记录页和事件列表联调到最新接口`  
动作：触发本 skill，先做 `D03/S01` 契约对齐，再实现页面与状态。

**示例 2**
输入：`后台导出任务列表要补失败重试和状态提示`  
动作：触发本 skill，优先补齐错误态、重试流和验收检查。

**示例 3**
输入：`H5 分享页要支持过期态和脱敏展示`  
动作：触发本 skill，按安全与分享约束实现令牌过期/脱敏呈现。
